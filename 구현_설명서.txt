  1. 프로젝트 개요
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  이 시스템은 카메라로 방 안을 촬영하고 Vision Language Model(VLM)로 영상을 분석한 뒤,
  ISO 7730 국제표준의 PMV(Predicted Mean Vote, 예측 평균 온열감) 모델을 통해
  재실자가 느끼는 온열 쾌적성을 계산하여, 공조기(에어컨)를 자동으로 제어하는 시스템이다.

  최종 목표 하드웨어: NVIDIA Jetson Orin Nano Super (8GB)
  최종 목표 모델   : Qwen2.5-VL 3B (INT4 양자화 + TensorRT 가속)
  현재 개발 환경   : Windows 노트북 (CPU only), Qwen2-VL-2B-Instruct 모델

  2. 에어컨 시뮬레이터를 사용한 이유
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [이유 1] 실제 에어컨과의 통신 인터페이스가 없다
    실제 에어컨을 소프트웨어로 제어하려면 RS-485, IR 블래스터, 스마트홈 API 등이 필요하다.
    현재 단계에서는 이런 하드웨어 인터페이스가 없으므로 소프트웨어 시뮬레이터로 대체한다.

  [이유 2] 실내 온습도 센서도 없다
    PMV 계산에는 실내 기온(ta)과 습도(rh)가 필수다.
    실제 IoT 센서(DHT22, SHT31 등)가 없어도 시스템 전체를 테스트할 수 있도록
    시뮬레이터가 '현재 실내 온도'와 '현재 실내 습도'를 계산하여 제공한다.

  [이유 3] 알고리즘 검증이 우선이다
    VLM 분석 → PMV 계산 → 제어 명령 생성 파이프라인이 올바르게 동작하는지
    먼저 소프트웨어 레벨에서 검증한 뒤, 추후 Jetson + 실제 센서/에어컨으로 포팅할 계획이다.

  [결론]
  시뮬레이터가 하는 일은 크게 두 가지다.
    (A) 에어컨 역할: 설정 온도/풍량에 따라 실내 온도를 낮추고 제습한다.
    (B) 실내 센서 역할: 현재 실내 온도와 습도 값을 계산하여 제공한다.
  실제 시스템에서는 (A)는 에어컨 API로, (B)는 IoT 센서로 교체된다.
  교체 시 sensor_interface.py 만 수정하면 된다.

  4-1. weather_service.py  [외부 날씨 데이터 수집]
  ────────────────────────────────────────────────────────────────

  역할:
    기상청 초단기실황 API (api.data.go.kr)에 HTTP GET 요청을 보내
    현재 위치의 실외 기상 데이터를 수신한다.
    위경도 좌표를 기상청 격자(nx, ny)로 자동 변환하여 사용한다.
    API 호출이 실패할 경우 마지막 성공값 또는 기본값(20°C, 50%)을 반환한다.

  API 엔드포인트:
    http://apis.data.go.kr/1360000/VilageFcstInfoService_2.0/getUltraSrtNcst

  수신 항목:
  ┌──────────────┬────────────┬──────────────────────────────────────────────┐
  │  변수명       │  단위      │  설명                                        │
  ├──────────────┼────────────┼──────────────────────────────────────────────┤
  │  out_temp    │  °C        │  현재 실외 기온  (카테고리: T1H)             │
  │  out_humid   │  %         │  현재 실외 상대 습도  (REH)                  │
  │  out_wind    │  m/s       │  실외 풍속  (WSD)                            │
  │  out_weather │  문자열    │  강수 형태 (PTY): 맑음/비/비눈/눈/소나기     │
  └──────────────┴────────────┴──────────────────────────────────────────────┘

  기본값 (API 실패 시):
    out_temp = 20.0°C, out_humid = 50.0%, out_weather = "unknown", out_wind = 0.0

  API 키 설정:
    .env 파일에 WEATHER_API_KEY=<공공데이터포털 인증키> 를 추가한다.
    키가 없으면 기본값을 반환하며 시스템은 계속 동작한다.

  4-3. vlm_processor.py  [VLM 비전 분석]
  ────────────────────────────────────────────────────────────────

  역할:
    카메라 프레임을 Qwen2-VL-2B-Instruct 모델로 분석하여
    PMV 계산에 필요한 착의량(clo), 대사율(met), 공간 맥락 신호를 추출한다.
    ※ 인원 수는 YOLODetector가 전담 — VLM 프롬프트에서 제외됨.

  디바이스 우선순위:
    1. MPS  (Apple Silicon M-시리즈)  — float16
    2. CUDA (NVIDIA GPU)               — float16
    3. CPU  (그 외)                    — float32

  처리 과정:
    ① 프레임을 160×160으로 다운스케일 (CPU 부하 절감)
    ② BGR → RGB → PIL 이미지 변환
    ③ 모델에 JSON 형식 응답 요청:
       {"sleeves": "long"|"short",
        "outerwear": "yes"|"no",
        "activity": "lying"|"sitting"|"standing"|"walking"|"cooking"|"exercising",
        "room_size": "small"|"medium"|"large",
        "heat_source": "yes"|"no"}
    ④ 응답에서 정규표현식으로 JSON 추출 후 파싱
    ⑤ PMV 입력값으로 변환 (아래 매핑 참조)

  VLM 출력 → PMV 파라미터 매핑:

    [착의량 (Clo) 매핑]  - ISO 7730:2005 기준
    ┌────────────────────────────────┬──────────┐
    │  VLM 분류                       │  Clo 값  │
    ├────────────────────────────────┼──────────┤
    │  sleeves=short                 │  0.5     │
    │  sleeves=long                  │  1.0     │
    │  outerwear=yes 시 추가          │  +0.3    │
    └────────────────────────────────┴──────────┘
    예시: 긴팔 + 아우터 → Clo = 1.0 + 0.3 = 1.3

    [대사율 (Met) 매핑]  - ISO 7730:2005 기준
    ┌────────────────────────────────┬──────────┐
    │  activity 값                    │  Met 값  │
    ├────────────────────────────────┼──────────┤
    │  lying      (누워있음)          │  0.8     │
    │  sitting    (착석)              │  1.0     │
    │  standing   (기립)              │  1.2     │
    │  walking    (보행)              │  1.5     │
    │  cooking    (조리)              │  2.0     │
    │  exercising (운동)              │  3.0     │
    │  인식 불가 시 기본값            │  1.2     │
    └────────────────────────────────┴──────────┘

    [공간 크기 매핑]
    small  → 15 m²,  medium → 30 m²,  large → 60 m²

    [열원 감지 시 복사온도 보정]
    heat_source=yes → tr = ta + 4.0°C  (조리기구 등 복사열 반영)

  반환값:
    {"clo": float, "met": float, "activity": str,
     "room_size": str, "room_size_m2": float,
     "heat_source": str, "outerwear": str}
    분석 실패 시 None 반환 → 메인 루프에서 기본값으로 유지

  생성 변수:
  ┌─────────────┬────────┬───────────────────────────────────────────────────┐
  │  변수명      │ 단위   │  설명                                             │

  4-6. hvac_simulator.py  [에어컨 + 실내 환경 시뮬레이터]
  ────────────────────────────────────────────────────────────────

  역할:
    (A) 가상 에어컨: 설정된 목표 온도와 풍량에 따라 실내 온도를 변화시킨다.
    (B) 가상 실내 센서: 매 스텝마다 실내 온도/습도를 계산하여 제공한다.
    실제 배포 시 (A)는 에어컨 API로, (B)는 DHT22/SHT31 같은 IoT 센서로 교체된다.

  내부 상태 변수:
  ┌──────────────┬────────┬──────────────────────────────────────────────────┐
  │  변수명       │ 단위   │  설명                                            │
  ├──────────────┼────────┼──────────────────────────────────────────────────┤
  │  indoor_temp │  °C    │  현재 실내 공기 온도 (초기값: 22.0°C)           │
  │  indoor_humid│  %     │  현재 실내 상대 습도 (초기값: 50.0%)            │
  │  room_size   │  m²    │  방 면적 (기본값: 20.0m²)                       │
  │  window_open │  bool  │  창문 개폐 여부 (True=열림, False=닫힘)          │
  │  is_on       │  bool  │  에어컨 전원 상태                               │
  │  target_temp │  °C    │  에어컨 설정 온도 (기본값: 24.0°C)             │
  │  fan_speed   │  1~3   │  풍량 (1=약, 2=중, 3=강)                       │
  │  mode        │  문자열│  운전 모드 ('cool' 또는 'heat')                 │
  └──────────────┴────────┴──────────────────────────────────────────────────┘

  simulate_step() 물리 모델 (30fps 기준):

    [Step 1] 방 크기 보정계수
      size_factor = 20.0 / max(room_size, 5.0)
      → 방이 기준(20m²)보다 클수록 온도 변화가 느려진다.

    [Step 2] 에어컨 냉·난방
      냉방: indoor_temp -= 0.001 × fan_speed × size_factor   ≈ 1.8°C/min (Fan3)
      난방: indoor_temp += 0.0008 × fan_speed × size_factor  ≈ 1.4°C/min (Fan3)

    [Step 3] 외기 온도 평형
      equil_rate = 0.005 (창문 열림) 또는 0.0003 (창문 닫힘)
      indoor_temp += (outdoor_temp - indoor_temp) × equil_rate

    [Step 4] 창문 열릴 때 외부 습도 영향
      indoor_humid += (outdoor_humid - indoor_humid) × 0.01

    [Step 5] 재실 인원 체열
      indoor_temp += people_count × 0.0003 × size_factor   ≈ 0.54°C/min (1인)

    [Step 6] 물리적 범위 클램핑
      indoor_temp  = clamp(10.0 ~ 45.0)
      indoor_humid = clamp(15.0 ~ 95.0)

  4-7. thermal_engine.py  [ISO 7730 PMV 열쾌적성 계산]
  ────────────────────────────────────────────────────────────────

  역할:
    ISO 7730:2005 및 ASHRAE 55 표준 기반 PMV(Predicted Mean Vote) 계산.
    PMV는 사람이 느끼는 온열감을 -3(매우 춥다) ~ +3(매우 덥다)으로 표현한다.

  입력 파라미터:
  ┌──────┬──────────────────────┬────────┬──────────────────────────────────┐
  │ 변수 │  의미                 │ 단위   │  출처                            │
  ├──────┼──────────────────────┼────────┼──────────────────────────────────┤
  │  ta  │  실내 공기 온도       │  °C    │  hvac_simulator.indoor_temp      │
  │  tr  │  평균 복사 온도       │  °C    │  ta (열원 감지 시 +4°C 보정)     │
  │  rh  │  실내 상대 습도       │  %     │  hvac_simulator.indoor_humid     │
  │  vel │  실내 기류 속도       │  m/s   │  fan_speed → FAN_VELOCITY        │
  │  met │  대사율               │  met   │  VLM 또는 Motion 감지 결과       │
  │  clo │  착의량               │  clo   │  VLM 분석 결과                   │
  └──────┴──────────────────────┴────────┴──────────────────────────────────┘

  풍량 → 기류 속도 (control_logic.py의 FAN_VELOCITY):
    fan_speed = 1 (약) → vel = 0.1 m/s
    fan_speed = 2 (중) → vel = 0.3 m/s
    fan_speed = 3 (강) → vel = 0.5 m/s

  내부 계산: ISO 7730:2005 Appendix C 수식 구현
    - 착의 표면 온도(tcl) 반복 수렴 (최대 150회, 허용 오차 0.0001°C)
    - 6개 열손실 항목 계산 (피부·발한·호흡잠열·호흡현열·복사·대류)
    - NaN/Inf 방어: 비정상 입력 시 0.0 반환

  PMV → 쾌적감 분류:
  ┌────────────────────┬────────────────────────────────┐
  │  PMV 범위           │  상태                          │
  ├────────────────────┼────────────────────────────────┤
  │  +2.5 초과          │  매우 더움 (Hot)               │
  │  +1.5 ~ +2.5        │  더움 (Warm)                   │
  │  +0.5 ~ +1.5        │  조금 더움 (Slightly Warm)     │
  │  -0.5 ~ +0.5        │  쾌적 (Neutral)  ← 목표       │
  │  -1.5 ~ -0.5        │  조금 추움 (Slightly Cool)     │
  │  -2.5 ~ -1.5        │  추움 (Cool)                   │
  │  -2.5 미만          │  매우 추움 (Cold)               │
  └────────────────────┴────────────────────────────────┘

  4-8. pid_controller.py  [PMV 기반 PID 제어기]

  4-9. control_logic.py  [PMV 히스테리시스 제어 + 창문 판단]
  ────────────────────────────────────────────────────────────────

  역할:
    main.py와 scenario_runner.py가 공유하는 순수 함수 모듈.

  decide_control(pmv_val, people_count, pid, hvac_is_on, hvac_mode, dt)
    목표온도 = 항상 24°C 고정.
    인원 = 0    → 에어컨 OFF, PID 초기화
    PMV > +0.5  → 냉방 ON  (팬 속도: PID 출력)
    냉방 중 PMV > +0.2  → 냉방 유지  (히스테리시스)
    PMV < -0.5  → 난방 ON  (팬 속도: PID 출력)
    난방 중 PMV < -0.2  → 난방 유지  (히스테리시스)
    쾌적 구간   → OFF, PID 초기화

  decide_window(pmv_val, outdoor_temp, indoor_temp, heat_source, hvac_mode, people_count)
    인원 = 0            → 창문 닫기
    heat_source = yes   → 창문 열기 (환기 우선)
    난방 모드            → 현 상태 유지
    PMV > 0.5 이고 실외 온도 < 실내 - 2°C  → 창문 열기 (자연환기)
    그 외               → 현 상태 유지

    10:00 | 실내  25.1°C | PMV +0.31 | 꺼짐 Fan1 | 인원 6명 | [STEADY]   →  PMV +0.31 — 쾌적, 에어컨 OFF
    17:00 | 실내  24.8°C | PMV +0.47 | 꺼짐 Fan1 | 인원 6명 | [STEADY]   →  PMV +0.47 — 쾌적, 에어컨 OFF
    18:00 | 실내  24.6°C | PMV  0.00 | 꺼짐 Fan1 | 인원 0명 | [EMPTY]    →  공실 감지 — 에어컨 OFF

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  9. 환경 오버라이드 / 수동 제어 기능
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [환경 오버라이드 (E키)]
    시뮬레이터 물리 계산 결과를 덮어쓰고 특정 환경값을 강제로 고정한다.
    실내온도 35°C로 설정 → PMV 급상승 → 냉방 ON 등 제어 반응을 즉시 확인 가능.
    대시보드에 보라색 "ENV OVERRIDE" 섹션이 나타나 현재 강제값을 표시한다.
    'e' 키로 해제하면 시뮬레이터 물리 계산으로 복귀한다.

  조작 순서:
    e   → 오버라이드 ON (현재 시뮬레이터 값으로 초기화)
    [/] → 조정할 변수 선택 (실내온도 / 실외온도 / 실내습도 / 실외습도)
    +/- → 선택된 변수 값 ±1°C (온도) 또는 ±5% (습도)
    e   → 오버라이드 OFF

  [수동 제어 (M키)]
    자동 PMV 제어 대신 직접 에어컨 설정을 지정한다.
    대시보드에 빨간색 "수동 조작 중" 표시가 나타난다.
    다시 m 키를 누르면 자동 모드로 복귀하며 PID가 초기화된다.

  조작 순서:
    m     → 수동 모드 ON (현재 에어컨 상태 복사)
    p     → 전원 ON/OFF 토글
    c / h → 냉방 / 난방 모드
    +/-   → 설정온도 ±1°C (16~30°C 범위)
    f     → 팬 속도 순환 (1→2→3→1)
    m     → 자동 모드 복귀
