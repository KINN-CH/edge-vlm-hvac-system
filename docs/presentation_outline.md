## SLIDE 3 — 연구 배경 및 문제 정의

**제목:** 왜 기존 HVAC는 부족한가?

**핵심 메시지 (3가지):**

```
❶ 기존 HVAC = 온도 센서만 의존
   → "지금 몇 도인가"만 알고, "누가, 얼마나 더운가"는 모름

❷ 실내 쾌적성은 온도 하나로 결정되지 않음
   → 착의량(Clo), 활동량(Met), 인원 수, 복사열 등 복합 요소

❸ 클라우드 AI는 사용 불가
   → 보안 시설, 네트워크 없는 환경, 개인정보 문제
```

**보조 내용:**
- 기존 온도 제어 방식: 설정온도 ±0.5°C 구간에서 ON/OFF 반복
- PMV 기준 쾌적 구간(-0.5 ~ +0.5)을 유지하는 시스템은 시중에 없음
- 사생활 보호: 카메라 영상이 외부로 나가지 않아야 함

**[발표 멘트]**
기존 에어컨은 온도 센서 하나에 의존합니다. 그러나 실제 인체가 느끼는
쾌적함은 온도 외에도 착의량, 활동량, 인원 수에 따라 크게 달라집니다.
저희 프로젝트는 이 문제를 카메라 기반 AI로 해결하되,
완전 오프라인으로 동작시키는 것이 핵심입니다.

## SLIDE 5 — 제안 시스템 개요

**제목:** 시스템 한 줄 요약

```
카메라 + 날씨 API + YOLO
        ↓
   PMV 열쾌적성 계산 (ISO 7730)
        ↓
   PID 제어기 → 에어컨 자동 제어
```

**3개 입력 소스:**

```
┌─────────────────────────────────────────────────────┐
│  VLM (Qwen2-VL-2B)     → 착의량, 운동량, 방 크기    │
│  YOLO (YOLOv8n)         → 인원 수                   │
│  외부 날씨 API           → 외부 온도, 습도, 날씨     │
└─────────────────────────────────────────────────────┘
              ↓ 융합
        ISO 7730 PMV 계산
              ↓
        PID 제어 → 에어컨
```

**타겟 하드웨어:** NVIDIA Jetson Orin Nano Super (8GB)
**개발 환경:** 노트북 CPU / Apple Silicon M5 (MPS)

## SLIDE 7 — 핵심 기술 ① : VLM 기반 공간 맥락 인지

**제목:** Vision Language Model — 공간을 이해하는 AI

**모델:** Qwen2-VL-2B-Instruct (완전 오프라인)
**분석 주기:** 30초 (CPU) / 10초 (Apple Silicon) / 5초 (Jetson TRT)

**VLM이 추출하는 정보:**

```
카메라 프레임 (160×160 다운스케일)
           ↓  Qwen2-VL-2B 추론
    ┌──────────────────────────────┐
    │ {                            │
    │   "sleeves": "long",         │  → clo = 1.0
    │   "outerwear": "yes",        │  → clo += 0.3  →  clo = 1.3
    │   "activity": "sitting",     │  → met = 1.0 (ISO 7730)
    │   "room_size": "medium",     │  → 30 m²
    │   "heat_source": "no"        │  → tr 보정 없음
    │ }                            │
    └──────────────────────────────┘
```

**착의량(Clo) 계산:**
```
소매 길이:  short → 0.5 clo  /  long → 1.0 clo
아우터 착용: yes  → +0.3 clo
최종 clo  = sleeves_clo + outerwear_clo
```

**활동량(Met) 매핑 (ISO 7730:2005):**
```
lying(0.8) → sitting(1.0) → standing(1.2)
→ walking(1.5) → cooking(2.0) → exercising(3.0)
```

**디바이스 자동 선택:** MPS(Apple) → CUDA(NVIDIA) → CPU 순 자동 감지

**폴백 구조:**
```

```

```
프레임 차분 → motion_score (0.0 ~ 1.0)
→ motion_score 높으면 MET를 motion 기반으로 실시간 보정
→ 갑작스러운 운동량 변화도 즉각 반영
```

## SLIDE 9 — 핵심 기술 ③ : ISO 7730 PMV 열쾌적성 계산

**제목:** PMV — 사람이 느끼는 더위/추위를 숫자로

**PMV (Predicted Mean Vote):** ISO 7730:2005 국제 표준
```
범위: -3 (매우 추움) ────── 0 (쾌적) ────── +3 (매우 더움)
쾌적 구간: -0.5 ≤ PMV ≤ +0.5
```

**PMV 계산 입력 변수:**

| 변수 | 단위 | 출처 |
|------|------|------|
| ta (공기 온도) | °C | SensorInterface (시뮬레이터) |
| tr (복사 온도) | °C | ta + 4°C (열원 있을 때) |
| rh (습도) | % | SensorInterface (시뮬레이터) |
| vel (풍속) | m/s | Fan1=0.1 / Fan2=0.3 / Fan3=0.5 |
| met (대사율) | met | VLM activity 매핑 |
| clo (착의량) | clo | VLM sleeves + outerwear 계산 |

**PMV 상태 분류:**
```
PMV < -1.5  : 매우 추움 → 난방 최대
-1.5 ~ -0.5 : 추움      → 난방 가동
-0.5 ~ +0.5 : 쾌적      ← 목표 구간
+0.5 ~ +1.5 : 더움      → 냉방 가동
PMV > +1.5  : 매우 더움 → 냉방 최대
```

**[발표 멘트]**
PMV는 단순히 온도계를 보는 것이 아니라, 사람의 옷차림과 활동 강도까지
반영해서 "지금 사람이 얼마나 덥고 춥게 느끼는가"를 -3에서 +3 사이로
표현하는 ISO 국제 표준입니다.

## SLIDE 10 — 핵심 기술 ④ : PID 제어기

**제목:** PMV 오차를 0으로 만드는 PID 제어기

**목표:** PMV = 0.0 유지 (열 중립 상태)

**PID 수식:**
```
error      = 0.0 - PMV_현재값

output = kp × error
       + ki × ∫error dt     (누적 오차 보정)
       + kd × d(error)/dt   (급격한 변화 억제)
```

**파라미터:**
```
kp = 0.8   (비례 이득)  — 현재 오차에 즉각 반응
ki = 0.05  (적분 이득)  — 오래 지속된 오차 누적 보정
kd = 0.3   (미분 이득)  — 오버슈트 방지
deadband = 0.12         — |output| < 0.12 → 변경 없음 (미세 진동 방지)
```

**출력 해석:**
```
output > 0  →  난방 필요 (PMV < 0, 너무 추움)
output < 0  →  냉방 필요 (PMV > 0, 너무 더움)
output = 0  →  현재 상태 유지

|output| 0.0~1.0  →  Fan 1 (0.1 m/s)
|output| 1.0~2.0  →  Fan 2 (0.3 m/s)
|output| 2.0~3.0  →  Fan 3 (0.5 m/s)
```

**[발표 멘트]**
STEADY 상태에서는 PMV를 0으로 만드는 것이 목표입니다.
PID 제어기는 단순한 if/else 온도 임계값 제어와 달리, 오차가 얼마나
오래 지속됐는지(ki), 얼마나 빠르게 변하는지(kd)까지 고려하므로
오버슈트 없이 안정적으로 쾌적 구간에 수렴합니다.

## SLIDE 11 — 핵심 기술 ⑤ : 시계열 상태 머신

**제목:** 공간 맥락을 시간 흐름으로 추적

**4단계 상태:**
```
EMPTY  ──────────────▶  ARRIVAL  ──────────────▶  STEADY
(공실)   인원 감지 시       (도착)   10분 경과 후     (안정)
  ▲                                                   │
  │  인원 0, 30분 지속                    퇴근 점수 ≥ 55
  │                                                   ▼
  └──────────────────────────────────  PRE_DEPARTURE
                                         (퇴근 준비)
```

**각 상태의 제어 방식:**
```
EMPTY         →  HVAC OFF, PID 리셋
ARRIVAL       →  규칙 기반 강제 냉난방 (Fan3, 10분간)
                 - 외부 < 15°C : 난방 25°C
                 - 외부 > 28°C : 냉방 22°C
STEADY        →  PID 제어 (PMV 기반 자동)
PRE_DEPARTURE →  Fan 1 절전 모드
```

**퇴근 맥락 점수 계산 (최대 75점):**
```
인원 감소 추세  : +30점
아우터 착용     : +25점   (VLM 감지)
기립 자세       : +10점   (VLM 감지)
퇴근 시간대     : +10점   (18시 ± 1시간)
───────────────────────
진입 임계값     : 55점 이상  →  PRE_DEPARTURE
복귀 임계값     : 30점 미만  →  STEADY 복귀 (오탐 방지)
```

## SLIDE 13 — 에너지 절감 효과

**제목:** 정량적 에너지 절감 측정

**베이스라인 비교 방식:**
```
기존 방식 (Baseline)  :  항상 Fan 2 상시 가동
본 시스템             :  상태 기반 가변 제어

절약률 = (baseline_kWh - actual_kWh) / baseline_kWh × 100%
```

**예상 절감 시나리오:**
```
공실(EMPTY) 구간  →  HVAC OFF  →  베이스라인 대비 100% 절약
PRE_DEPARTURE    →  Fan 1     →  베이스라인 대비 ~60% 절약
STEADY + PMV 쾌적  →  유지만   →  베이스라인 대비 ~30% 절약
```

**CSV 자동 기록 항목:**
```
timestamp, system_state, people_count, pmv_val, comfort_status,
fan_speed, power_w, energy_kwh, baseline_kwh, savings_pct, comfort_rate
```

**쾌적율(Comfort Rate):**
```
PMV ∈ [-0.5, +0.5] 구간 유지 비율 (%)
→ 논문 증빙 자료로 직접 활용
```

> 그래프 삽입: 시간축 PMV 변화 / 누적 에너지 절약률 곡선

## 부록 B — Q&A 예상 질문

**Q1. VLM이 사람의 옷을 잘못 인식하면 어떻게 되나요?**
A. VLM 분석은 30초 주기이며, 이전 값을 유지하는 폴백이 있습니다.
   또한 PID 제어기의 deadband(0.12) 덕분에 미세한 오차는 제어에 반영되지 않습니다.

**Q2. 사람이 잠깐 자리를 비우면 에어컨이 꺼지나요?**
A. 인원 0 감지 후 30초 확인 대기가 있으며, 퇴근 시간대가 아닌 경우
   departure_score가 낮게 유지됩니다. 이 문제는 향후 단계적 부재 감지 로직으로 개선 예정입니다.

**Q3. 실제 에어컨과 어떻게 연결하나요?**
A. 현재는 소프트웨어 시뮬레이터로 검증 중입니다. Jetson 이전 시
   RS-485 또는 GPIO 기반 제어 모듈로 교체할 예정입니다.

**Q4. Qwen2-VL 모델 크기가 큰 것 아닌가요?**
A. 2B 파라미터 모델로 약 4~8GB 메모리를 사용합니다.
   Jetson(8GB)에서 INT4 양자화 시 2GB 이하로 줄어들어 실시간 운용이 가능합니다.
