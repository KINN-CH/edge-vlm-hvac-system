================================================================================
  엣지 VLM HVAC 제어 시스템 — 알고리즘 및 변수 명세
  (Edge VLM Intelligent HVAC Control System — Algorithm & Variable Specification)
================================================================================
최종 수정: 2026-04-03
--------------------------------------------------------------------------------

1. 시스템 개요
================================================================================

카메라(VLM) + 외부 날씨 API + YOLO 인원 감지 세 가지 입력 소스를 융합하여
ISO 7730 PMV(예측 평균 온열감) 지수를 실시간으로 계산하고,
PID 제어기를 통해 HVAC(냉난방공조) 장치를 자동 제어합니다.

  [ 입력 소스 ]
  ┌──────────────────┬──────────────────────────────────────────────────────┐
  │ VLM (Qwen2-VL)   │ 착의량(clo), 운동량(met/activity), 방 크기(room_size) │
  │ YOLO (YOLOv8n)   │ 인원 수 (people_count)                               │
  │ 외부 날씨 API     │ 외부 온도, 습도, 날씨 상태, 풍속 (기상청 KMA)         │
  │ 대기질 API       │ PM10, PM2.5, 통합대기환경지수 (에어코리아)             │
  └──────────────────┴──────────────────────────────────────────────────────┘

  [ 처리 흐름 ]
  입력 수집 → 상태 머신 갱신 → PMV 계산 → PID 제어 → HVAC 명령 출력


2. 입력 소스별 변수 명세
================================================================================

2-1. VLM (Vision Language Model) — Qwen2-VL-2B-Instruct
   분석 주기: 30초 (CPU 기본) / 10~15초 (MPS) / 5~10초 (Jetson TRT)
   입력 해상도: 160×160 픽셀 (CPU 부하 절감용 다운스케일)

   ┌──────────────┬───────────────────────────────┬───────────────────────┐
   │ VLM 출력 키  │ 가능한 값                     │ 용도                   │
   ├──────────────┼───────────────────────────────┼───────────────────────┤
   │ sleeves      │ 'long' / 'short'              │ 기본 착의량(clo) 산출  │
   │ outerwear    │ 'yes' / 'no'                  │ clo 보정 (+0.3)        │
   │ activity     │ lying / sitting / standing /  │ 대사율(met) 산출       │
   │              │ walking / cooking / exercising│                        │
   │ room_size    │ 'small' / 'medium' / 'large'  │ 유효 열부하 면적 결정  │
   │ heat_source  │ 'yes' / 'no'                  │ 복사온도(tr) 보정      │
   └──────────────┴───────────────────────────────┴───────────────────────┘

   착의량 계산 (Clothing Insulation, clo):
     clo = CLO_BASE[sleeves]              → short: 0.5 / long: 1.0
     if outerwear == 'yes': clo += 0.3   → 아우터 착용 시 +0.3 clo

   대사율 계산 (Metabolic Rate, met):
     ┌────────────┬───────────────────────────────────────┐
     │ activity   │ met 값 (ISO 7730:2005 Table B.1 기준) │
     ├────────────┼───────────────────────────────────────┤
     │ lying      │ 0.8  (수면/완전 휴식)                 │
     │ sitting    │ 1.0  (사무 작업 착석)                 │
     │ standing   │ 1.2  (기립 / 가벼운 활동)             │
     │ walking    │ 1.5  (보행)                           │
     │ cooking    │ 2.0  (조리 / 서서 작업)               │
     │ exercising │ 3.0  (유산소 운동)                    │
     └────────────┴───────────────────────────────────────┘
     분류 불가 기본값: 1.2 (기립 수준)

   방 크기 추정 (Room Size):
     ┌──────────┬───────────┬──────────────────────────┐
     │ VLM 값   │ 환산 m²   │ 기준                      │
     ├──────────┼───────────┼──────────────────────────┤
     │ small    │ 15.0 m²   │ 소형 사무실 / 개인실       │
     │ medium   │ 30.0 m²   │ 중간 회의실 / 교실         │
     │ large    │ 60.0 m²   │ 강의실 / 대형 공간         │
     └──────────┴───────────┴──────────────────────────┘
     → HVACSimulator의 room_size 파라미터로 전달
       (방 크기는 HVAC 열부하 계산 시 면적 환산에 사용됨)

   열원 보정 (Heat Source Correction):
     if heat_source == 'yes': tr_corrected = indoor_temp + 4.0°C
     → 복사온도(tr)를 실제 온도보다 높게 보정하여 PMV 상향 반영

   MotionDetector 보정 (met_source 결정):
     MotionDetector는 매 프레임 프레임 차분으로 motion_score (0.0~1.0) 산출
     → motion_score > 임계값 시 MET를 motion 기반으로 오버라이드
     → met_source: 'motion' 또는 'vlm'

3. PMV 계산 알고리즘
================================================================================

PMV(Predicted Mean Vote): ISO 7730:2005 표준의 열 쾌적성 지수
범위: -3 (매우 추움) ~ 0 (중립/쾌적) ~ +3 (매우 더움)

입력 변수:
  ┌───────────┬───────────┬──────────────────────────────────────────────┐
  │ 변수       │ 단위      │ 출처 및 설명                                  │
  ├───────────┼───────────┼──────────────────────────────────────────────┤
  │ ta        │ °C        │ 실내 공기 온도 (SensorInterface → 시뮬레이터) │
  │ tr        │ °C        │ 복사온도 = ta (열원 있으면 ta + 4.0°C 보정)   │
  │ rh        │ %         │ 실내 습도 (SensorInterface → 시뮬레이터)      │
  │ vel       │ m/s       │ 공기 유속 = FAN_VELOCITY[fan_speed]           │
  │           │           │   Fan 1: 0.1 m/s / Fan 2: 0.3 m/s           │
  │           │           │   Fan 3: 0.5 m/s                             │
  │ met       │ met       │ 대사율 (VLM activity → met 매핑, 또는 motion) │
  │ clo       │ clo       │ 착의량 (VLM sleeves + outerwear → clo 계산)   │
  └───────────┴───────────┴──────────────────────────────────────────────┘

쾌적 상태 분류:
  ┌────────────────┬────────────────────────────────┐
  │ PMV 범위        │ 상태                           │
  ├────────────────┼────────────────────────────────┤
  │ -0.5 ≤ PMV ≤ 0.5 │ 쾌적 (Comfortable)          │
  │ -1.0 ≤ PMV < -0.5 │ 약간 추움 (Slightly Cool)  │
  │  0.5 < PMV ≤ 1.0 │ 약간 더움 (Slightly Warm)   │
  │ -2.0 ≤ PMV < -1.0 │ 추움 (Cool)                │
  │  1.0 < PMV ≤ 2.0 │ 더움 (Warm)                 │
  │ PMV < -2.0       │ 매우 추움 (Very Cold)        │
  │ PMV > 2.0        │ 매우 더움 (Very Hot)         │
  └────────────────┴────────────────────────────────┘


4. PID 제어기
================================================================================

목표: PMV = 0.0 (열 중립 상태 유지)

파라미터:
  ┌─────────────┬────────┬──────────────────────────────────────────────┐
  │ 파라미터     │ 값     │ 역할                                         │
  ├─────────────┼────────┼──────────────────────────────────────────────┤
  │ PMV_TARGET  │ 0.0    │ 목표 PMV (ISO 7730 열 중립)                   │
  │ kp          │ 0.8    │ 비례 이득 — 현재 PMV 오차에 즉각 반응          │
  │ ki          │ 0.05   │ 적분 이득 — 지속적 오차 누적 보정             │
  │ kd          │ 0.3    │ 미분 이득 — 급격한 변화 억제 (오버슈트 방지)  │
  │ DEADBAND    │ 0.12   │ |output| < 0.12 → 출력 0 (미세 진동 방지)    │
  │ INTEGRAL_MAX│ 10.0   │ 적분 항 클램핑 (anti-windup)                 │
  │ OUTPUT_MAX  │ 3.0    │ 최대 출력 클램핑                              │
  └─────────────┴────────┴──────────────────────────────────────────────┘

PID 출력 계산:
  error      = PMV_TARGET - pmv_val         (PMV 오차)
  integral  += error × dt                   (오차 누적, anti-windup 적용)
  derivative = (error - prev_error) / dt    (오차 변화율)
  output     = kp×error + ki×integral + kd×derivative
  if |output| < DEADBAND: output = 0        (데드밴드 처리)
  output     = clamp(output, -3.0, +3.0)

PID 출력 → 팬 속도 변환:
  ┌────────────────────┬───────────────────────┐
  │ |output| 범위       │ 팬 속도                │
  ├────────────────────┼───────────────────────┤
  │ 0.0 ~ 1.0          │ Fan 1 (0.1 m/s)       │
  │ 1.0 ~ 2.0          │ Fan 2 (0.3 m/s)       │
  │ 2.0 ~ 3.0          │ Fan 3 (0.5 m/s)       │
  └────────────────────┴───────────────────────┘

출력 부호 해석:
  output > 0 → 난방 필요 (PMV가 목표보다 낮음, 즉 너무 추움)
  output < 0 → 냉방 필요 (PMV가 목표보다 높음, 즉 너무 더움)
  output = 0 → 현재 설정 유지 (쾌적 구간)


5. 상태 머신 (State Machine)
================================================================================

4단계 상태로 공간 맥락을 추적하여 제어 모드를 전환합니다.

상태 정의:
  ┌──────────────┬───────────────────────────────────────────────────────┐
  │ 상태          │ 의미 및 제어 방식                                     │
  ├──────────────┼───────────────────────────────────────────────────────┤
  │ EMPTY        │ 빈 공간 — HVAC OFF, PID 리셋                          │
  │ ARRIVAL      │ 도착 직후 — 규칙 기반 강제 냉난방 (Fan 3, 10분간)     │
  │              │   out_temp < 15°C → 난방 25°C / Fan 3                 │
  │              │   out_temp > 28°C → 냉방 22°C / Fan 3                 │
  │              │   그 외        → 온도 기반 모드 선택 / Fan 2           │
  │ STEADY       │ 안정 운전 — PID 제어기 (PMV 기반 팬속도 자동 조절)    │
  │ PRE_DEPARTURE│ 퇴근 준비 감지 — Fan 1 절전, 제어 최소화              │
  └──────────────┴───────────────────────────────────────────────────────┘

상태 전이 조건:
  EMPTY        → ARRIVAL       : 인원 > 0 감지
  ARRIVAL      → STEADY        : ARRIVAL 진입 후 600초 경과
  STEADY       → PRE_DEPARTURE : 퇴근 맥락 점수 ≥ 55
  PRE_DEPARTURE → STEADY       : 퇴근 점수 < 30 (오탐 복귀)
  PRE_DEPARTURE → EMPTY        : 인원 = 0
  ANY          → EMPTY         : 인원 0 상태 30초 지속 확인

퇴근 맥락 점수 계산 (최대 75점):
  ┌──────────────────────────────┬─────┬───────────────────────────────┐
  │ 신호                          │ 점수 │ 조건                         │
  ├──────────────────────────────┼─────┼───────────────────────────────┤
  │ 인원 감소 추세                │ +30 │ count < 이전 분석 count        │
  │ 아우터 착용                   │ +25 │ VLM outerwear == 'yes'        │
  │ 기립 자세                     │ +10 │ VLM activity == 'standing'    │
  │ 퇴근 시간대                   │ +10 │ work_end_hour ± 1시간 내      │
  └──────────────────────────────┴─────┴───────────────────────────────┘
  PRE_DEPARTURE 진입 임계값: 55점 (아우터+인원감소 동시 조건 충족 필요)
  STEADY 복귀 임계값: 30점

  ※ 가방(bags) 신호는 VLM 인식 정확도 한계로 제외.

7. 에너지 모니터링
================================================================================

베이스라인 비교: 전통 방식(항상 Fan 2 가동) 대비 실제 소비전력 차이로 절약률 산출

  ┌─────────────────┬───────────────────────────────────────────────────┐
  │ 지표             │ 의미                                              │
  ├─────────────────┼───────────────────────────────────────────────────┤
  │ power_w         │ 현재 소비전력 (W)                                  │
  │ energy_kwh      │ 누적 소비 에너지 (kWh)                             │
  │ baseline_kwh    │ 베이스라인 누적 에너지 (kWh)                       │
  │ savings_pct     │ 절약률 (%) = (baseline - actual) / baseline × 100 │
  │ comfort_rate    │ PMV ∈ [-0.5, 0.5] 구간 유지 비율 (%)              │
  └─────────────────┴───────────────────────────────────────────────────┘

  CSV 로그 파일: hvac_system_performance.csv
  → 누적 데이터로 에너지 절감 효과 논문 증빙 자료 사용


8. 하드웨어 디바이스 자동 선택
================================================================================

  MPS (Apple Silicon M1~M5) → float16  (우선순위 1)
  CUDA (NVIDIA GPU)          → float16  (우선순위 2)
  CPU (그 외)                → float32  (우선순위 3)

  Jetson 이전 시 변경 사항:
  - sensor_interface.py: MODE = 'simulate' → 'dht22' 또는 'bme280'
  - yolo_detector.py: imgsz=320 → 640, TensorRT 엔진 사용
  - VLM: Qwen2-VL INT4 + TensorRT-LLM 변환 (convert_tensorrt.py --all)
  - 분석 주기: --interval 5 (5~10초)
