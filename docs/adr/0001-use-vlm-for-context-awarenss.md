# ADR-0001: VLM 기반 공간 맥락 인지 시스템 도입

## Status

Accepted

---

## Context

기존 HVAC 제어 시스템은 온도 및 습도 센서와 같은 정량 데이터에만 의존하여 동작한다.
이 방식은 다음과 같은 한계를 가진다:

* 활동량(Met), 착의량(Clo) 등 쾌적성 핵심 요소 측정 불가
* 입실/퇴실 등 상황(Context) 기반 제어 불가능
* 가방 소지, 열원(조리기구) 등 비정형 맥락 감지 불가

또한, 본 프로젝트는 **완전 오프라인 환경**에서 동작해야 하므로
클라우드 기반 AI 사용이 제한되는 제약이 존재한다.

이러한 문제를 해결하기 위해
**비정형 데이터(이미지)를 활용한 상황 인지 방식**이 필요하다.

---

## Decision

카메라 입력을 기반으로 **VLM(Qwen2-VL-2B-Instruct)** 을 활용하여
공간의 맥락(Context)을 인지하는 시스템을 도입한다.

VLM이 담당하는 항목 (정성적 맥락 인지):

* 착의량 — sleeves(long/short) + outerwear → **Clo 값 산출**
* 활동 분류 — lying/sitting/standing/walking/cooking/exercising → **Met 값 산출**
* 가방 소지 여부 (bags) → **퇴근 맥락 점수**에 반영
* 열원 존재 여부 (heat_source) → **복사온도(tr) 보정** 및 환기 우선 제어

> **인원 수(People Count)는 VLM이 담당하지 않는다.**
> VLM의 인원 감지 정확도가 불안정(~60~70%)하여,
> 별도 결정(ADR-0002)에 따라 **YOLOv8n** 이 전담한다.

VLM은 30초 주기 백그라운드 스레드로 비블로킹 실행되며,
분석 결과는 Queue를 통해 메인 루프에 전달된다.

---

## Consequences

### 장점

* 단순 센서 기반 대비 높은 표현력 확보
* 비정형 데이터(행동, 상황) 처리 가능
* 사용자 쾌적성(PMV) 예측 정확도 향상
* 오프라인 환경에서도 지능형 제어 가능
* 인원 감지를 YOLO에 위임하여 VLM 프롬프트 단순화 → 추론 속도 개선

### 단점

* 연산 비용 증가 (Jetson 리소스 부담, CPU 추론 30~60초/회)
* 모델 최적화 필요 — INT4 양자화 + TensorRT-LLM (`convert_tensorrt.py` 참고)
* VLM 착의량·활동 분류 정확도에 의존

---

## 📌 Purpose (목적)

이 ADR 문서는 다음을 목적으로 한다:

* **지식 전수:** VLM 도입 배경과 이유를 명확히 공유
* **반복 논쟁 방지:** 동일한 기술 선택에 대한 재논의 방지
* **온보딩 가속화:** 신규 팀원이 빠르게 시스템 이해 가능
