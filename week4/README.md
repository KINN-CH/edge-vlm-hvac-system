# [Week 4] GitHub Flow 및 코드리뷰 협업 실습

## 🌿 브랜치 전략

GitHub Flow 방식으로 feature 브랜치를 생성하여 작업 후 PR을 통해 main에 병합합니다.

## 📝 Conventional Commits 적용

| Type | 예시 |
|------|------|
| `feat` | `feat(vlm): add prompt engineering for Met/Clo output` |
| `fix` | `fix(hvac): resolve temperature convergence error` |
| `docs` | `docs: add CONTRIBUTING guide` |

## 🔍 PR 리뷰 태그 시스템

| 태그 | 설명 |
|------|------|
| `[MUST]` | 반드시 수정 필요 |
| `[SHOULD]` | 수정 강력 권장 |
| `[NITS]` | 사소한 제안 |
| `[QUESTION]` | 코드 의도 확인 |
| `[PRAISE]` | 잘 작성된 코드 칭찬 |

## 🗂 산출물

- `.github/pull_request_template.md` — PR 템플릿
- `.github/CODEOWNERS` — 코드 소유자 설정
- `CONTRIBUTING.md` — 기여 가이드
- Branch Protection 설정 완료
